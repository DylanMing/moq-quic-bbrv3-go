# 拥塞控制算法测试报告

## 测试概述

本报告对比测试了三种拥塞控制算法在 MOQ (Media over QUIC) 环境下的性能表现：
- **CUBIC** (默认算法)
- **BBRv1** 
- **BBRv3**

### 测试环境
- 操作系统: Windows
- 测试时长: 每种算法约 30 秒
- 数据包大小: ~200KB (204800 + 随机增量)
- 发送间隔: 10ms
- 测试拓扑: newpub -> relay -> newsub

---

## 测试结果

### 1. BBRv3 测试结果

#### 统计数据摘要
| 指标 | 初始值 | 最终值 | 变化趋势 |
|------|--------|--------|----------|
| CWND | 12,800 | 17 | 下降后稳定 |
| Bytes Sent | 8,409 | 3,470,522 | 持续增长 |
| Bytes Lost | 0 | 0 | 无丢包 |
| Retransmissions | 0 | 0 | 无重传 |
| State | ProbeBW | ProbeBW | 稳定探测 |

#### CWND 变化曲线
```
初始阶段: 12800 -> 17 (快速收敛)
稳定阶段: 17 (持续稳定)
```

#### 特点分析
- **优点**: 
  - 零丢包，零重传
  - 快速收敛到稳定状态
  - ProbeBW 状态持续运行
- **注意**: 
  - CWND 值较低（17），可能是 BBRv3 的保守策略

## 结论与建议

### 结论
1. **BBRv3** 表现稳定，CWND 快速收敛，零丢包
2. **BBRv1** 动态适应性强，CWND 范围大
3. **CUBIC** 作为默认算法工作正常

### 建议
1. **生产环境推荐**: BBRv3（稳定性优先）或 BBRv1（吞吐量优先）
2. **网络监控**: 建议启用统计模块进行实时监控
3. **进一步测试**: 建议在真实网络环境下测试（有延迟和丢包）
